Drag the link below to your bookmarks bar, then click it on any X/Twitter thread page.

📥 Scrape Thread

Usage

  1. Open a thread on x.com or twitter.com.
  2. Click the Scrape Thread bookmarklet — a panel appears bottom-right.
  3. Click Auto-scroll. It scrolls to top, then down, capturing tweets and queuing linked threads.
  4. Click Process queue to crawl every queued link sequentially — a single worker tab opens, loads each page, scrapes it, and moves on. Click Stop to halt.
  5. Use Rescan page to re-capture the current page (handy after Reset all).
  6. Click Download JSON (or Copy JSON) when done.

Notes

  • Hooks fetch / XHR at install time — the initial page-load request was already consumed, but scrolling up to the thread root re-fetches it.
  • Stores URLs (pfp, media), not base64, by default. Toggle Capture pfp base64 in the panel to inline profile pictures as data URIs in the export.
  • Reply structure: each tweet has in_reply_to_status_id and conversation_id.
  • Process queue reuses one worker tab, navigating it from page to page (no tab storm). Allow pop-ups for the site once.
